# 参与贡献 fishhook

我们希望让参与贡献这个项目的过程尽可能简单和透明。

## Pull Request

我们非常欢迎你提交 Pull Request。

1. Fork 本仓库并从 `master` 创建你的分支。
2. 如果你添加了需要测试的代码，请同步添加相应的测试用例。
3. 如果你修改了 API，请同步更新相关文档。
4. 确保所有测试均能通过。
5. 确保你的代码通过代码规范检查（lint）。
6. 如果你尚未签署贡献者许可协议（CLA），请先完成签署。

## 贡献者许可协议（CLA）

为了能够接受你的 Pull Request，我们需要你提交一份 CLA。对于 Facebook 的所有开源项目，你只需签署一次即可。

请在此处完成 CLA 签署：<https://code.facebook.com/cla>

## Issues（问题反馈）

我们使用 GitHub Issues 来追踪公开的 Bug。请确保你的描述清晰明确，并提供足够的复现步骤。

Facebook 设有[漏洞奖励计划](https://www.facebook.com/whitehat/)，专门用于安全漏洞的负责任披露。对于安全相关的 Bug，请按照该页面所述流程进行报告，不要直接提交公开的 Issue。

## 许可证

向 fishhook 提交贡献，即表示你同意你的贡献将依据本源码树根目录下的 LICENSE 文件所规定的许可证进行授权。
